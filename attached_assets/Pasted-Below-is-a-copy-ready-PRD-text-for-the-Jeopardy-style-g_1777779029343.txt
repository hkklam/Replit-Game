Below is a copy-ready PRD text for the Jeopardy-style game. This version focuses on the game/product requirements, with clear instructions for the AI developer to build the MVP first, then Version 2, and to check for bugs.

⸻

PRD: Jeopardy-Style Multiplayer Quiz Game

1. Product Name

Quiz Board Showdown

A browser-based Jeopardy-style quiz game where players choose categories and point values, answer trivia questions, wager points, and compete to finish with the highest score.

This game should be inspired by the format of category-based trivia games, but it must use original questions and original branding. Do not use the Jeopardy name, logo, music, graphics, or copyrighted clues.

⸻

2. Product Goal

Create a fun, fast, multiplayer trivia game that can be played online by families, students, friends, and casual players.

The first version should be simple and reliable:

* Up to 4 online players
* A game board with categories and point values
* Players choose clues
* Players buzz in or answer in turn
* Scores update automatically
* Final wager round
* Built-in question bank
* Clean game-show style interface

⸻

3. Target Users

Primary Users

* Families
* Kids and students
* Teachers
* Trivia fans
* Friends playing online
* Casual game night players

Age Range

Recommended age: 8+

The game should support both kid-friendly and general trivia questions.

⸻

4. Core Game Concept

Players compete on a trivia board with multiple categories. Each category has point-value clues. Higher-value clues are harder.

Players select a clue, read the question, and try to answer correctly. Correct answers earn points. Incorrect answers lose points or allow other players to answer.

The game ends after all questions are completed or after a selected number of rounds. The player with the highest score wins.

2. Science & Nature

12. Question Difficulty Rules

100-Point Questions

Should be easy and widely known.

Example:

Question: What planet is known as the Red Planet?
Answer: Mars

200-Point Questions

Should require basic school-level knowledge.

Example:

Question: What gas do plants absorb from the air during photosynthesis?
Answer: Carbon dioxide

300-Point Questions

Should require stronger general knowledge.

Example:

Question: What ancient civilization built the pyramids at Giza?
Answer: Ancient Egypt

400-Point Questions

Should be harder, with less obvious answers.

Example:

Question: What is the longest river in South America?
Answer: The Amazon River

500-Point Questions

Should be challenging.

Example:

Question: What physicist developed the general theory of relativity?
Answer: Albert Einstein

14. Game Flow

Full Game Flow

1. Player opens game
2. Player chooses Create Room or Join Room
3. Host creates room
4. Other players join with room code
5. Host starts game
6. Game board appears
7. First player chooses category and point value
8. Question appears
9. Players answer
10. Score updates
11. Question tile becomes unavailable
12. Next player chooses
13. Continue until board is complete
14. Final wager round begins
15. Players submit wagers
16. Final question appears
17. Players submit answers
18. Final scores are calculated
19. Winner screen appears

20. Answer Modes

Mode A: Turn-Based Answering

Simpler for MVP.

Flow:

1. Current player selects question
2. Only current player answers
3. Correct answer earns points
4. Wrong answer loses points if negative scoring is enabled
5. Turn passes to next player

This is the recommended MVP mode because it is easier to build and less likely to have multiplayer timing bugs.

Mode B: Buzz-In Answering

Better for Version 2 or later MVP improvement.

Flow:

1. Question is shown to all players
2. Buzz button becomes active
3. First player to buzz gets answer attempt
4. Correct answer earns points
5. Wrong answer loses points and locks that player out
6. Other players may buzz
7. If nobody answers correctly, reveal answer

⸻

21. Recommended MVP Answer Mode

Use turn-based answering for the first MVP.

Reason:

* Easier to build
* Easier to test
* Fewer real-time bugs
* Works better with slow internet
* More suitable for kids and casual players

After MVP is stable, add buzz-in mode.

22. Scoring Rules

Standard Scoring

Result	Score Change
Correct answer	Add tile points
Wrong answer	Subtract tile points
Pass / no answer	0
Final correct answer	Add wager
Final wrong answer	Subtract wager

Optional Easy Mode

For younger players:

Result	Score Change
Correct answer	Add tile points
Wrong answer	0
Pass	0

The host should be able to turn negative scoring on or off.

⸻

23. Timer Rules

Timer options:

* Off
* 15 seconds
* 30 seconds
* 45 seconds

When timer expires:

* Answer is marked as no answer
* No points are awarded
* Turn passes to next player

If buzz-in mode is used:

* Timer starts when question appears
* Players must buzz before time expires
* Answering player gets a shorter answer timer

⸻

24. Final Wager Round

The final round should work like this:

1. Show final category
2. Each player secretly enters a wager
3. Wager cannot exceed their current score
4. If player has negative score, wager is 0
5. Show final question
6. Players submit answers secretly
7. Reveal answers
8. Correct answer adds wager
9. Incorrect answer subtracts wager
10. Highest score wins

MVP Simplification

For MVP, if secret answer submission is too complicated, use a simpler final round:

* Each player enters wager
* Each player answers one at a time
* Reveal correct answer
* Update scores

25. Daily Double-Style Bonus Tile

To avoid copying directly, call it:

Power Pick

Power Pick rules:

* Hidden under one random board tile
* Only the player who selects it can answer
* Player can wager some or all points
* If correct, they gain wager
* If wrong, they lose wager

MVP Option

Do not include Power Pick in the first MVP unless the base game is already stable.

Add it in Version 2.

28. Core User Stories

Host User Stories

As a host, I want to create a game room so that my friends can join.

As a host, I want to choose game settings so that I can make the game easier or harder.

As a host, I want to start the game when all players are ready.

As a host, I want the game to automatically track scores so I do not need to calculate manually.

Player User Stories

As a player, I want to join a room with a code so I can play with friends.

As a player, I want to choose a category and point value so I can control the game strategy.

As a player, I want to type my answer so the system can check if I am correct.

As a player, I want to see my score and other players’ scores.

As a player, I want to know whose turn it is.

As a player, I want to see the final winner.

* Empty player names are rejected

* Turns rotate correctly
* Used questions cannot be selected again
* Final winner is calculated correctly

⸻

32. Game Logic Bug Checklist

Question Selection

Check that:

* A player can select only available tiles
* Used tiles become disabled
* Correct question appears
* Category and point value match selected tile
* Question cannot be opened twice

Answer Checking

Check that:

* Correct answer is accepted
* Wrong answer is rejected
* Case differences do not matter
* Extra spaces do not matter
* Acceptable alternate answers work
* Empty answer is not accepted
* Timer expiration works

Example answer normalization:

function normalizeAnswer(answer) {
  return answer
    .toLowerCase()
    .trim()
    .replace(/[^\w\s]/g, "")
    .replace(/\s+/g, " ");
}

Scoring

Check that:

* Correct answer adds points
* Wrong answer subtracts points if enabled
* Wrong answer gives 0 if negative scoring is disabled
* Scores can go negative if negative scoring is enabled
* Final wager adds or subtracts correctly
* Player cannot wager more than current score

Turn Rotation

Check that:

* Turn moves to next player after each question
* Disconnected players are skipped
* Host leaving does not crash game
* Current player is highlighted
* Non-current players cannot select clues in turn-based mode

⸻

33. Multiplayer Bug Checklist

Check that:

* All players see the same board
* All players see the same scores
* All players see the same current turn
* Tile selection updates for everyone
* Answer result updates for everyone
* Reconnecting player receives current game state
* Two players cannot select tiles at the same time
* Server controls score changes
* Client cannot fake score updates
* Client cannot answer when it is not their turn

⸻

34. Final Round Bug Checklist

Check that:

* Final round starts after board is complete
* Players can enter valid wagers
* Negative-score players cannot wager
* Wagers are hidden from other players until reveal
* Final answers are submitted correctly
* Correct answers add wager
* Wrong answers subtract wager
* Final winner is calculated correctly
* Ties are handled

Tie handling:

39. Edge Cases

Situation	Expected Behavior
Player disconnects	Mark disconnected and skip turn after timeout
Host disconnects	Transfer host to next player
Room is full	Show room full message
Invalid room code	Show invalid room code message
Player refreshes page	Attempt reconnect
Two players click same tile	Server accepts first valid click only
Question bank too small	Show error and prevent game start
Duplicate question IDs	Log error and ignore duplicate
Empty answer	Ask player to enter answer
All questions answered	Start final round or end game
Tie score	Show tie result

* Players can answer questions
* Scores update correctly

* Used tiles cannot be reused
* Game ends when board is complete
* Final round works
* Winner is shown
* Game works on desktop and mobile
* No major console errors occur
* No player can cheat by changing client-side score
* Question bank loads correctly

45. MVP Summary

The MVP should be a simple but complete game:

A player creates a room.
Up to 4 players join.
The host starts the game.
Players take turns choosing category tiles.
They answer trivia questions.
Scores update automatically.
After the board is complete, players enter a final wager.
The final question is answered.
The player with the highest score wins.

The game should be fun, clear, stable, and easy to expand.
